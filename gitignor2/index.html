reset --soft HEAD~1 - это не эта команда

git commit --amend --no-edit

  • 1. пушим наши изменения на GitHub
  • 2. Делаем следующие изменения и понимаем, что они должны быть в п.1
  • 3. git add . = добавляет изменения в индекс (staging area).
  • 4. git commit --amend --no-edit создает новый коммит, включая в него изменения, и сохраняет существующее сообщение коммита (git commit -m 'feat: add point one').
  • 5. git push origin ваша_ветка --force проталкивает изменения на удаленный репозиторий, используя опцию --force, так как вы изменили историю коммитов.
